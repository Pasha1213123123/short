# 🐛 Завдання: Виправлення логіки реклами (Ads Fix)

## 📌 Опис проблеми
Зараз інтеграція реклами працює не ідеально:
1. **Логіка показів:** Реклама повинна запускатися строго після кожного 3-го свайпу (навіть якщо користувач просто швидко пролистує відео, це рахується). Поточна роздільна логіка з Nswipe і Nfinish конфліктує і працює не так, як очікується. Потрібен єдиний лічильник свайпів.
2. **Проблема з плеєром (Пауза):** Під час показу повноекранної реклами відео на задньому плані продовжує відтворювати звук і грати. Відео має ставати на жорстку паузу перед показом реклами і відновлювати відтворення **тільки після** того, як користувач закриє рекламу.

## 🛠 План вирішення (Roadmap)

### 1. Глобальний стан реклами (Riverpod)
Щоб відео гарантовано не грало, нам потрібен глобальний стейт, який "глушить" плеєр.
* Створити `isAdShowingProvider = StateProvider<bool>((ref) => false);`.
* В `ShortPlayerScreen` додати слухача: якщо `isAdShowing == true`, викликати `controller.pause()`. Це заблокує будь-які спроби відео грати під час реклами.

### 2. Спрощення лічильника (Тригер 3-х свайпів)
* Видалити лічильник завершень відео (`_finishCounter`). Залишити лише один лічильник свайпів (`_swipeCounter`).
* Лічильник `_swipeCounter` збільшується на **кожен** перехід (свайп) у `_onPageChanged`, незалежно від напрямку чи швидкості.
* Коли `_swipeCounter == 3`:
  1. Встановлюємо `isAdShowingProvider = true` (відео ставиться на паузу).
  2. Викликаємо `AdsManager.showInterstitialAd()`.
  3. Чекаємо на закриття реклами (через `await`).
  4. Встановлюємо `isAdShowingProvider = false` (відео знімається з паузи і може грати далі, якщо autoplay увімкнено).
  5. Скидаємо лічильник `_swipeCounter = 0`.

### 3. Оновлення `AdsManager`
* Метод `showInterstitialAd` повинен коректно повертати `Future`, який завершується **тільки після закриття реклами** користувачем, а не одразу після виклику методу `show()`. Для цього можна використати `Completer<bool>`.
