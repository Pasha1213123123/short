# 🗺 Roadmap: Інтеграція Interstitial Реклами (Ads)

Цей документ описує етапи впровадження повноекранної реклами в стрічку Shorts з використанням гібридного підходу (AdMob + AppLovin) та двох незалежних тригерів.

## 📌 Загальна концепція
*   **Рекламні мережі:** AdMob (основна), AppLovin MAX (резервна/fallback).
*   **Тригер A (Nswipe=5):** Показ після кожних 5 свайпів (змін сторінок).
*   **Тригер B (Nfinish=3):** Показ після завершення перегляду кожних 3 відео.
*   **Поведінка:** Пауза відео під час реклами, автоматичне відновлення після закриття.

---

## 🟢 Етап 1: Підготовка та Конфігурація (Environment)
1.  **Залежності (`pubspec.yaml`):**
    *   Додати `google_mobile_ads: ^5.1.0`.
    *   Додати `applovin_max: ^2.6.0`.
2.  **Налаштування Платформ:**
    *   **Android:** Додати `applovin.sdk.key` та перевірити AdMob App ID у `AndroidManifest.xml`.
    *   **iOS:** Оновити `Info.plist` (SKAdNetworkItems, AppLovin Key, AdMob ID).
3.  **Секрети (`.env`):**
    *   Додати ідентифікатори рекламних блоків (Ad Units) для обох мереж для Android та iOS.

## 🟡 Етап 2: Сервісний шар (Service Layer)
1.  **Створення `AdsManager` (`lib/services/ads_manager.dart`):**
    *   Метод `initialize()`: Ініціалізація обох SDK.
    *   Механізм Preload: Завантаження оголошень заздалегідь для мінімізації затримок.
    *   Метод `showInterstitialAd()`:
        *   Логіка Fallback: Спочатку AdMob, якщо не готовий — AppLovin.
        *   Блокування (`isAdShowing`): Запобігання подвійним показам.
        *   Повернення `Future<bool>` для підтвердження успішного показу.

## 🟠 Етап 3: Інтеграція в UI та Логіку Стрічки
1.  **Глобальна Ініціалізація:**
    *   Виклик ініціалізації реклами в `main.dart` перед `runApp`.
2.  **Стан Стрічки (`lib/screens/shorts_page_view.dart`):**
    *   Додати лічильники: `_swipeCounter`, `_finishCounter`.
    *   Додати прапорець `_suppressNextPageAsSwipe` (щоб програмний перехід не рахувався як свайв).
3.  **Впровадження Тригерів:**
    *   **Тригер A:** Оновлення `_onPageChanged` для інкременту `_swipeCounter`.
    *   **Тригер B:** Оновлення колбеку `onVideoFinished` у `ShortPlayerScreen` для інкременту `_finishCounter`.
4.  **Керування Плеєром:**
    *   Використання `currentVideoControllerProvider` для виклику `pause()` перед рекламою та `play()` після закриття.

## 🔵 Етап 4: Аналітика та Тестування
1.  **Логування:**
    *   Додати події Firebase Analytics: `ad_attempt`, `ad_shown_admob`, `ad_shown_applovin`, `ad_failed`.
2.  **Тест-кейси:**
    *   Перевірка "швидких свайпів" (чи не ламається черга).
    *   Перевірка перемикання жанрів (чи скидаються лічильники, якщо потрібно, або чи продовжують рахувати).
    *   Перевірка роботи при вимкненому інтернеті (відсутність реклами не повинна блокувати контент).
