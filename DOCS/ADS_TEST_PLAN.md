# 🧪 План тестування: Інтеграція реклами (Ads)

Цей документ описує всі необхідні перевірки для підтвердження коректної роботи рекламних тригерів, логіки Fallback та взаємодії з відеоплеєром.

## 1. Unit-тести: Логіка `AdsManager`
Ці тести перевіряють внутрішню логіку сервісу без запуску UI.

| ID | Опис | Очікуваний результат |
|:---|:---|:---|
| **U1.1** | Ініціалізація SDK | При виклику `initialize()` ініціалізуються обидва SDK (AdMob та AppLovin). |
| **U1.2** | Логіка Fallback (AdMob ready) | Якщо AdMob завантажений, `showInterstitialAd()` викликає показ саме AdMob. |
| **U1.3** | Логіка Fallback (AdMob fail, AppLovin ready) | Якщо AdMob порожній, а AppLovin готовий, показується AppLovin. |
| **U1.4** | Блокування подвійного показу | Якщо `isAdShowing == true`, повторний виклик `showInterstitialAd()` повертає `false`. |
| **U1.5** | Перезавантаження (Reload) | Після закриття реклами (dismissed) автоматично запускається запит на нове оголошення. |

## 2. Widget-тести: Інтеграція в Ленту (ShortsPageView)
Перевірка лічильників та тригерів у взаємодії з UI.

| ID | Опис | Очікуваний результат |
|:---|:---|:---|
| **W2.1** | Тригер A (Свайпи) | Після 5-го свайпу вперед викликається `showInterstitialAd()`. |
| **W2.2** | Напрямок свайпу | Свайп назад (newIndex < oldIndex) не збільшує `_swipeCounter`. |
| **W2.3** | Тригер B (Завершення відео) | Після 3-го виклику `onVideoFinished` реклама показується ПЕРЕД переходом на наступну сторінку. |
| **W2.4** | Скидання лічильників | При зміні фільтра жанрів `_swipeCounter` та `_finishCounter` скидаються в 0. |
| **W2.5** | Програмна навигація | Перехід на наступну сторінку через `nextPage()` не повинен рахуватися як "свайв користувача" (suppression). |

## 3. Функціональні тести: Взаємодія з відео (Player Control)
Перевірка того, як реклама впливає на перегляд контенту.

| ID | Опис | Очікуваний результат |
|:---|:---|:---|
| **F3.1** | Пауза при показі | В момент виклику реклами поточне відео ставиться на паузу. |
| **F3.2** | Відновлення після закриття | Після закриття реклами відео продовжує грати (якщо увімкнено Autoplay). |
| **F3.3** | Відсутність реклами (No Fill) | Якщо жодна мережа не віддала рекламу, користувач просто переходить до наступного відео без затримок. |
| **F3.4** | Стан звуку | Реклама не повинна залишати відео без звуку після закриття, якщо звук був увімкнений. |

## 4. Граничні випадки (Edge Cases)

| ID | Опис | Очікуваний результат |
|:---|:---|:---|
| **E4.1** | Відсутність інтернету | Додаток не зависає, реклама ігнорується, контент доступний. |
| **E4.2** | Швидкі свайпи | Якщо користувач свайпає дуже швидко, лічильник рахує коректно, але реклама не "нашаровується". |
| **E4.3** | Останнє відео в списку | Якщо 3-тє завершення відео припадає на останній ролик, реклама показується, але `nextPage` не викликається (бо нема куди). |
| **E4.4** | Помилка показу (Show Fail) | Якщо реклама завантажена, але не змогла показатися, відео має автоматично знятися з паузи. |

## 5. Аналітика та Логи

| ID | Опис | Очікуваний результат |
|:---|:---|:---|
| **L5.1** | `ad_interstitial_attempt` | Подія відправляється в Firebase при кожній спробі показу. |
| **L5.2** | `ad_interstitial_success` | Подія відправляється лише при успішному показі. |
| **L5.3** | Debug Logs | В консолі видно: "Swipe Counter: X" та "Finish Counter: Y" для зручності налагодження. |
