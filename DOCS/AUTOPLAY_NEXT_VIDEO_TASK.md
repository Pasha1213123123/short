# ▶️ Автовоспроизведение следующего ролика

Отдельная задача по реализации функции автоматического перехода к воспроизведению следующего ролика после окончания текущего.

---

## Задача

**AUTOPLAY-001**: Функция авто-воспроизведения следующего ролика

- **Описание:** После окончания текущего видео приложение должно автоматически переключаться на следующий ролик в ленте и начинать его воспроизведение без действия пользователя.
- **Контекст:** Shorts-формат предполагает непрерывный просмотр: конец ролика → следующий ролик.
- **Приоритет:** Улучшение (UX), важно для формата short-form video.

### Требования

- По завершении воспроизведения текущего ролика (достижение конца или 100%) автоматически перейти к следующему ролику в `PageView` / списке.
- Запустить воспроизведение следующего ролика без паузы (или с минимальной задержкой при предзагрузке).
- Учитывать настройку «Autoplay» в настройках приложения (`lib/screens/settings_screen.dart`): при выключенном Autoplay авто-переход можно не выполнять или оставить только переход без автозапуска — по договорённости в продукте.
- Не ломать существующее поведение: свайп вручную, pull-to-refresh, кэш и предзагрузка (`VideoCacheManager`, `ShortsPageView`).

### Реализация (подсказки)

**Место в коде**

- Экраны/виджеты: `lib/screens/shorts_page_view.dart` (PageView.builder, логика переключения страниц).
- Плеер: виджет/экран воспроизведения одного ролика (например, `ShortPlayerScreen` или встроенный плеер в ячейке PageView) — нужен слушатель окончания видео.
- Настройки: `lib/screens/settings_screen.dart` — опция Autoplay; значение хранить в `SharedPreferences` или провайдере (например, Riverpod) и читать при принятии решения об авто-переходе.

**Подход**

1. **Слушатель окончания видео**  
   В месте, где управляется `VideoPlayerController` текущего ролика, подписаться на завершение воспроизведения (например, `video_player` — слушатель позиции/длительности или `player.addListener` и проверка `position >= duration`).

2. **Действие при окончании**  
   При срабатывании «ролик закончился»:
   - Проверить, включён ли Autoplay (настройки).
   - Если да — вызвать переход на следующий индекс в `PageView` (например, через `PageController.nextPage` или аналог) и инициировать воспроизведение следующего ролика.

3. **Интеграция с PageView**  
   В `ShortsPageView` уже используется `PageView.builder` — переход на следующую страницу должен обновлять индекс и подставлять следующий элемент из списка видео; предзагрузка (если есть) должна учитывать следующий индекс.

4. **Опционально**  
   - Небольшая задержка перед авто-переходом (например, 0–300 мс), чтобы не было резкого скачка.  
   - Аналитика: событие `autoplay_next` (video_id текущего и следующего) для отчётов.

### Связанные файлы

- `lib/screens/shorts_page_view.dart` — список/лента роликов, PageView.
- `lib/screens/settings_screen.dart` — настройка Autoplay.
- Сервис/провайдер видеоплеера и кэша (например, `VideoCacheManager`, провайдеры из `lib/`).
