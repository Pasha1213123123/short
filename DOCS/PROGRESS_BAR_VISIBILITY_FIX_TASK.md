# 🐛 Завдання: Виправлення видимості повзунка прогресу (Progress Bar Fix)

## 📌 Опис проблеми
Після увімкнення режиму `edgeToEdge` для системного UI (який ми додали, щоб реклама перекривала весь екран), додаток почав малюватися "під" системною панеллю навігації (кнопки "Назад", "Додому" або смужка жестів унизу екрана). 
Через це повзунок прогресу відео (`Slider`), який жорстко прив'язаний до самого низу екрана (`bottom: 0`), виявився перекритим цією системною панеллю або зсунувся за межі видимої зони. Він стає видимим лише тоді, коли системна панель взаємодіє з користувачем.

## 🛠 План вирішення (Roadmap)

Щоб вирішити цю проблему, потрібно врахувати системні відступи (Safe Area), які повідомляють нам, скільки пікселів знизу займає системна панель.

### 1. Використання SafeArea / MediaQuery для нижньої частини
У файлі `lib/screens/short_player_screen.dart` повзунок позиціонується так:
```dart
Positioned(
  bottom: 0,
  left: 0,
  right: 0,
  child: _buildInteractiveProgressBar(),
)
```
Ми оновимо цей код, додавши врахування `MediaQuery.of(context).padding.bottom`. Це автоматично підніме повзунок рівно на ту висоту, яку займає системна смужка жестів або кнопки навігації на конкретному пристрої.

### 2. Оновлення коду
Замість `bottom: 0` ми вкажемо `bottom: MediaQuery.of(context).padding.bottom`, що динамічно адаптується під будь-який телефон (Android з кнопками, Android з жестами, iPhone).
