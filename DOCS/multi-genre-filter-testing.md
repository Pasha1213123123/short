# 🧪 План тестирования: Multi-Genre Filter & Roadmap Consistency

Этот документ содержит подробный перечень тест-кейсов для проверки функционала мультивыбора жанров и обеспечения консистентности интерфейса согласно Roadmap.

---

## 1. Unit-тесты: Логика провайдеров (`lib/providers.dart`)

Проверка бизнес-логики фильтрации и управления состоянием.

### 1.1. `selectedGenresProvider` (Состояние выбора)
- [ ] **T1.1.1: Начальное состояние.** Проверить, что при запуске сет выбранных жанров пуст (`Set<String> {}`).
- [ ] **T1.1.2: Добавление жанра.** Проверить, что при добавлении 'Action' сет содержит ровно один элемент.
- [ ] **T1.1.3: Удаление жанра.** Проверить, что после добавления и последующего удаления жанра сет снова становится пустым.
- [ ] **T1.1.4: Множественный выбор.** Проверить, что сет корректно хранит несколько уникальных жанров (например, {'Comedy', 'Drama'}).

### 1.2. `availableGenresProvider` (Список доступных жанров)
- [ ] **T1.2.1: Сортировка.** Проверить, что список всегда начинается с `'All'`.
- [ ] **T1.2.2: Алфавитный порядок.** Проверить, что остальные жанры отсортированы по алфавиту после 'All'.
- [ ] **T1.2.3: Уникальность.** Проверить, что в списке нет дубликатов, даже если несколько видео имеют один и тот же жанр.
- [ ] **T1.2.4: Пустой список видео.** Проверить, что если видео нет, провайдер возвращает только `['All']`.

### 1.3. `filteredVideosProvider` (Логика фильтрации - OR)
- [ ] **T1.3.1: Логика "All" (Пустой выбор).** Если выбранных жанров нет (сет пуст), провайдер должен вернуть полный список видео.
- [ ] **T1.3.2: Логика OR (Один жанр).** Если выбрано 'Action', должны вернуться только те видео, у которых в списке `genres` есть 'Action'.
- [ ] **T1.3.3: Логика OR (Несколько жанров).** Если выбраны 'Action' и 'Comedy', должны вернуться видео, у которых есть **хотя бы один** из этих жанров (или оба).
- [ ] **T1.3.4: Отсутствие совпадений.** Если выбран жанр, которого нет ни в одном видео, список должен быть пустым.

## 2. Widget-тесты: Интерфейс ленты (`lib/screens/shorts_page_view.dart`)

Проверка взаимодействия пользователя с элементами управления.

### 2.1. Чипы жанров (`GenreFilterChip`)
- [ ] **T2.1.1: Отображение фильтров.** Проверить, что при нажатии на кнопку "Menu" появляется список чипов.
- [ ] **T2.1.2: Состояние "All" по умолчанию.** При первом запуске чип "All" должен быть визуально выбран (`selected: true`).
- [ ] **T2.1.3: Переключение жанра.** Нажатие на чип (например, 'Comedy') делает его выбранным, а чип "All" — не выбранным.
- [ ] **T2.1.4: Очистка через "All".** Если выбрано несколько жанров, нажатие на "All" должно снять выделение со всех остальных и выделиться само.
- [ ] **T2.1.5: Множественное выделение.** Проверить, что можно выделить несколько чипов одновременно (кроме "All").

### 2.2. Сброс индекса и PageView
- [ ] **T2.2.1: Сброс позиции.** Если пользователь проскроллил до 5-го видео и изменил фильтр, `PageView` должен автоматически переключиться на 0-ю страницу.
- [ ] **T2.2.2: Обновление количества страниц.** Если после фильтрации осталось 2 видео, `itemCount` у `PageView` должен стать равен 2.
- [ ] **T2.2.3: Прелоад.** Проверить, что при смене фильтра вызывается метод `preload` у `VideoCacheManager`.

## 3. Widget-тесты: Экран загрузки (`lib/screens/upload_screen.dart`)

Проверка формы загрузки и консистентности UX.

### 3.1. Выбор жанров в Upload
- [ ] **T3.1.1: Наличие заголовка.** Проверить, что отображается локализованный текст "Select Genres" (или "Виберіть жанри" / "Выберите жанры").
- [ ] **T3.1.2: Базовые жанры (Fallback).** Проверить, что если в ленте нет видео, отображаются жанры по умолчанию из `AppConstants.defaultGenres`.
- [ ] **T3.1.3: Исключение служебных тегов.** Проверить, что в списке выбора **отсутствуют** чипы 'All', 'Mux', 'API'.
- [ ] **T3.1.4: Множественный выбор.** Проверить, что можно выделить несколько чипов для загружаемого видео.
- [ ] **T3.1.5: Блокировка при загрузке.** Проверить, что чипы становятся неактивными (disabled), когда идет процесс `_isUploading`.

## 4. Регрессионные и Edge Case тесты

### 4.1. Навигация и Состояние (Задача MOVE-001)
